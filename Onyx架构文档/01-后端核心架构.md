# 后端核心架构

> [!info] 模块路径
> `backend/onyx/` — FastAPI 应用核心，包含入口、路由、中间件、配置、错误处理等基础设施。

---

## 一、应用入口 (`main.py`)

**文件**: `backend/onyx/main.py`

### 启动流程

```
FastAPI 实例化
    → 加载配置 (app_configs.py)
    → 注册中间件链
    → 挂载 API 路由
    → 注册生命周期事件 (startup/shutdown)
    → 启动 Uvicorn (:8080)
```

### 中间件链（按执行顺序）

1. **CORS 中间件** — 跨域配置
2. **请求日志中间件** — 记录请求/响应信息
3. **认证中间件** — JWT / API Key / OAuth Token 验证
4. **租户中间件** (EE) — 多租户识别与隔离
5. **指标中间件** — Prometheus 指标采集
6. **追踪中间件** — OpenTelemetry Span 注入

### API 路由挂载

路由定义在 `backend/onyx/server/` 目录下，按功能域组织：

| 路由模块 | 路径前缀 | 职责 |
|---------|---------|------|
| `server/onyx_api/` | `/api` | 核心业务 API |
| `server/documents/` | `/api/document` | 文档管理 |
| `server/query_and_chat/` | `/api/chat` | 聊天与搜索 |
| `server/manage/` | `/api/manage` | 管理 API |
| `server/features/` | `/api/features` | 特性开关 |
| `server/settings/` | `/api/settings` | 系统设置 |
| `server/api_key/` | `/api/api-key` | API Key 管理 |
| `server/pat/` | `/api/pat` | 个人访问令牌 |
| `server/kg/` | `/api/kg` | 知识图谱 |
| `server/federated/` | `/api/federated` | 联合搜索 |
| `server/metrics/` | `/metrics` | Prometheus 指标端点 |
| `server/token_rate_limits/` | 内部 | 令牌速率限制 |

### 生命周期事件

- **startup**: 数据库连接池初始化、Redis 连接、加载 LLM 配置、预热缓存
- **shutdown**: 优雅关闭连接池、刷新追踪数据

## 二、配置系统 (`configs/`)

**核心文件**: `backend/onyx/configs/app_configs.py` (~1185 行)

### 配置架构

所有配置通过环境变量驱动，使用 Pydantic `BaseSettings` 模式：

```python
class OnyxConfig(BaseSettings):
    # 应用基础
    APP_HOST: str = "0.0.0.0"
    APP_PORT: int = 8080
    API_PREFIX: str = "/api"

    # 数据库
    POSTGRES_HOST: str
    POSTGRES_PORT: int = 5432
    POSTGRES_USER: str
    POSTGRES_PASSWORD: str
    POSTGRES_DB: str
    POSTGRES_POOL_SIZE: int = 40

    # Redis
    REDIS_HOST: str = "localhost"
    REDIS_PORT: int = 6379

    # LLM
    GEN_AI_API_KEY: str | None
    GEN_AI_MODEL_VERSION: str

    # ... 几百个配置项
```

### 配置文件清单

| 文件 | 内容 |
|------|------|
| `app_configs.py` | 主配置 (~1185 行)，涵盖应用、数据库、Redis、Celery、LLM、索引、连接器等 |
| `constants.py` | 枚举常量 (~711 行)：`DocumentSource`(55+)、`AuthType`、`OnyxCeleryQueues`、`OnyxRedisLocks`(~30) |
| `chat_configs.py` | 聊天参数：最大块数(25)、混合搜索 alpha(0.5)、历史压缩策略 |
| `llm_configs.py` | LLM 配置：嵌入模型、维度(768)、批处理大小、提示缓存 |
| `embedding_configs.py` | 10 个基础嵌入模型 × 2 精度 = 20 种配置 (Cohere/OpenAI/Gemini/Voyage/Nomic/E5) |
| `agent_configs.py` | Agent 参数：检索命中数(15)、重排序(10)、深度研究超时 |
| `kg_configs.py` | 知识图谱：实体提取超时、聚类阈值(0.6/0.96) |
| `onyxbot_configs.py` | Slack Bot：重试(5)、QPM 限制、响应速率 |

### 特性门控 (`feature_flags/`)

- **实现方式**: 通过 `variable_functionality.py` 的 `fetch_versioned_implementation()` 函数
- **CE/EE 切换**: 根据环境变量 `ENTERPRISE_EDITION_ENABLED` 选择社区版或企业版实现
- **特性键**: `feature_flags_keys.py` 定义所有可用特性标志

## 三、统一错误处理 (`error_handling/`)

### OnyxError 体系

```python
# 统一异常基类
class OnyxError(Exception):
    def __init__(
        self,
        error_code: OnyxErrorCode,  # 语义化错误码
        detail: str = "",
        status_code_override: int | None = None,  # 上游动态状态码
    )

# 错误码定义 (error_codes.py)
class OnyxErrorCode(StrEnum):
    NOT_FOUND = "not_found"
    UNAUTHENTICATED = "unauthenticated"
    FORBIDDEN = "forbidden"
    BAD_REQUEST = "bad_request"
    BAD_GATEWAY = "bad_gateway"
    INVALID_API_KEY = "invalid_api_key"
    # ... 几十个错误码
```

### 全局异常处理器

```python
@app.exception_handler(OnyxError)
async def onyx_error_handler(request, exc):
    return JSONResponse({
        "error_code": exc.error_code,
        "detail": exc.detail,
    }, status_code=exc.status_code)
```

**设计原则**: 永远使用 `OnyxError` 而非 `HTTPException`。错误码集中定义，新增错误类型需先添加到 `error_codes.py`。

## 四、缓存层 (`cache/`)

### 双后端缓存

```
缓存接口 (CacheInterface)
    ├── RedisBackend  — 生产环境，高性能
    └── PostgresBackend — 备用方案，无 Redis 时
```

- **Redis 后端**: 支持设置过期时间、批量操作
- **Postgres 后端**: 使用 KV 表存储，兼容无 Redis 部署
- **工厂模式**: `get_cache()` 自动选择后端

## 五、可观测性 (`tracing/`)

### 追踪框架

```
tracing/framework/
    ├── processor_interface.py  — 处理器接口
    └── provider.py             — 追踪提供者管理

追踪处理器:
    ├── BraintrustTracingProcessor — Braintrust 集成
    ├── LangfuseTracingProcessor   — Langfuse 集成
    └── OpenInference              — OpenTelemetry 标准
```

### 指标系统

- **Prometheus**: `prometheus_fastapi_instrumentator` 自动采集 HTTP 指标
- **自定义指标**: LLM 调用延迟、Vespa 查询时间、Postgres 查询时间
- **日志配置**: 可通过环境变量启用详细日志（`LOG_ENDPOINT_LATENCY`, `LOG_VESPA_TIMING_INFORMATION` 等）

### 数据脱敏

`tracing/masking.py` 提供敏感数据脱敏，确保追踪数据不泄露用户信息。

## 六、文件存储 (`file_store/`)

### 存储抽象

```
FileStoreInterface (抽象基类)
    ├── S3FileStore — S3/MinIO 存储
    └── PostgresFileStore — 数据库存储
```

- **后端选择**: `FILE_STORE_BACKEND` 环境变量 (`s3` / `postgres`)
- **模型**: `FileStore` 数据库模型跟踪文件元数据
- **用途**: 连接器拉取的文件、用户上传文件

---

## 七、键值存储 (`key_value_store/`)

```
KeyValueStoreInterface (抽象基类)
    └── PostgresKeyValueStore — 基于 PostgreSQL 的 KV 存储
```

- **工厂**: `get_key_value_store()` 创建实例
- **用途**: 系统配置、连接器状态、运行时元数据

---

## 八、文件处理引擎 (`file_processing/`)

支持的文件格式解析：
- **文档**: PDF, DOCX, PPTX, XLSX, TXT
- **标记语言**: HTML, Markdown
- **网页**: 通过 `trafilatura` 提取正文
- **通用**: 通过 `unstructured` 库处理各种格式

处理流程: 文件下载 → 格式识别 → 内容提取 → 文本清洗 → 结构化输出
