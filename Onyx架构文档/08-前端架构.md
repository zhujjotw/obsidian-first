### 流式响应处理

```typescript
// 使用 Server-Sent Events 接收流式响应
const eventSource = new EventSource(`/api/chat/stream?...`);

eventSource.onmessage = (event) => {
    const data = JSON.parse(event.data);

    switch (data.type) {
        case "message_delta":
            // 追加 Token 到当前消息
            appendToken(data.content);
            break;
        case "citation":
            // 添加引用标注
            addCitation(data.citation);
            break;
        case "tool_call":
            // 显示工具调用状态
            showToolCall(data.tool);
            break;
        case "message_end":
            eventSource.close();
            break;
    }
};
```

## 八、认证流程 (前端)

```
1. 页面加载
    → AuthProvider 检查本地 Token
    → 有效: 加载用户信息 → 进入应用
    → 无效: 重定向到 /auth/login

2. 登录
    → POST /api/auth/login
    → 存储 JWT + Refresh Token (httpOnly cookie 或 localStorage)
    → 更新 userStore

3. OAuth 登录
    → 重定向到 OAuth Provider
    → 回调到 /auth/oauth/callback
    → 交换 Code → Token
    → 存储并跳转

4. Token 刷新
    → Axios/Fetch 拦截器检测 401
    → 使用 Refresh Token 获取新 Access Token
    → 重试原请求

5. 登出
    → 清除本地 Token
    → 重定向到登录页
```
