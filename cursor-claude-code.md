# Cursor + Claude Code 结合使用技巧（Agent 应用）

## 定位与分工
- Claude Code：复杂推理、方案设计、风险评审、测试策略。
- Cursor：快速落地、多文件修改、重构、批量替换、调试迭代。

核心原则：先想清楚再动手，或先做 MVP 再系统复盘，按任务切换角色。

## 两种高效流程
### 先想后改
1. Claude Code 输出：目标/约束/流程/工具清单/失败回退/评估指标。
2. Cursor 按清单实现：模块拆分、接口对接、日志与测试。
3. Claude Code 做 review：风险点、边界条件、缺测路径。
4. Cursor 修复并补测试。

### 先改后审
1. Cursor 先做 MVP。
2. Claude Code 做系统性 review。
3. Cursor 迭代完善。

## Agent 场景常用模板
### 设计提纲（给 Claude Code）
- 目标与用户：
- 核心流程（Happy Path）：
- 工具清单与调用条件：
- 失败路径与回退策略：
- 安全边界（注入、数据、权限）：
- 评估指标与可观测性：

### 实现清单（给 Cursor）
- 规划器 / 路由器
- 工具封装与超时/重试
- 记忆/存储
- 保护策略（过滤/脱敏/权限）
- 评估与测试

### Review 清单（Claude Code）
- 提示注入风险
- 工具调用边界/权限
- 失败路径与可恢复性
- 数据泄露与日志敏感信息
- 测试覆盖：无效输入/工具异常/空结果

## 常见坑与规避
- 只做功能不做边界：必须补“失败回退”。
- 缺少可观测性：为关键步骤加结构化日志。
- 工具过多无路由：明确调用条件，避免链路失控。

## 日常最小流程（可直接复用）
1. Claude Code 出 1 页方案 + 风险清单。
2. Cursor 按模块实现 + 快速联调。
3. Claude Code 做 review + 测试建议。
4. Cursor 收尾 + 文档。
