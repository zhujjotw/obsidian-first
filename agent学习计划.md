 12 周平台向学习路线（每周 6 小时）

  第 1–2 周：工程底座

  - 目标：稳定 API + 工程规范
  - 产出：一个最小服务（鉴权/限流/日志）
  - 学习点：HTTP/REST、错误码设计、日志规范、配置管理

  第 3–4 周：LLM 接入与评测

  - 目标：可控输出 + 基础评估
  - 产出：统一的 LLM 调用层 + JSON schema 输出
  - 学习点：提示模板化、输出约束、用例集 + 手工评分

  第 5–6 周：Agent 框架骨架

  - 目标：Plan-Act-Observe 的 Agent 主循环
  - 产出：支持工具注册/调用/超时/重试
  - 学习点：工具接口设计、参数校验、失败回滚

  第 7–8 周：记忆与状态管理

  - 目标：可复用的短期/长期记忆模块
  - 产出：对话状态 + 任务状态存储
  - 学习点：缓存策略、检索、上下文裁剪

  第 9–10 周：多工具编排

  - 目标：稳定的多工具链
  - 产出：至少 3 个真实工具 + 编排策略
  - 学习点：依赖排序、超时策略、结果融合

  第 11–12 周：平台化

  - 目标：把 Agent 当“产品能力”提供
  - 产出：工具管理后台雏形 + 评测/监控面板
  - 学习点：多租户/权限、成本统计、可观测性
