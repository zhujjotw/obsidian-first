
## 1. 执行摘要

## 2. 法律问题

### 2.1 相关限制

欧盟《人工智能法案》 AI Act Annex III 里明确高风险的生物特征场景包括：

- **远程生物识别系统**：指在远程空间、对不特定人群进行识别的系统（通常指 1:N）。
    
- **生物特征分类（Biometric Categorisation）**：根据生物特征推断人的种族、政治取向、工会成员身份、宗教信仰或性取向（属于高度敏感的推断）。
    
- **情感识别系统（Emotion Recognition）**：在工作场所或教育机构中使用 AI 识别人的情绪，被列为高风险（甚至在某些特定行政执法场景下被禁止）。
    

仅用于**确认特定自然人身份的生物特征验证（1:1 ）**明确被排除在远程生物识别高风险之外。

  

> 举例：在家庭环境中确认我是爸爸，从而展现出爸爸对应的推荐内容

> 1. **为什么它通常不属于“高风险”？**
>     

> - **身份验证的除外条款**：源代码明确提到，仅用于确认特定自然人身份的 **1:1 生物特征验证**（如声纹解锁、确认“你是爸爸吗”）被明确排除在远程生物识别高风险之外。
>     

> - **非公共/远程识别**：法案中定义的“高风险”通常指在远程空间、对**不特定人群**进行的识别（1:N）。你的场景是在私密的家庭环境中对**特定、已知**的成员（爸爸、妈妈、小孩）进行识别，这更接近于一种身份索引或个性化服务入口。
>     

> - **成熟商业案例参考**：市场上已有的方案（如三星 Bixby Voice ID）已在家庭环境（如冰箱、电视）中实现类似功能，根据用户的声纹识别身份并展示私人日历、照片流等，被视为空间智能化的合规应用。
>     

> 1. **什么情况下可能会触及“高风险”？**
>     

> 虽然识别“谁在说话”本身风险较低，但如果你在功能中加入了以下逻辑，则可能进入高风险甚至受限区域：

> - **集成情感识别**：如果你不仅识别身份，还通过声纹分析爸爸此时的“情绪”（如判断爸爸不开心并推送安慰内容），在某些特定场景（如办公或教育）这属于**高风险**。虽然家庭场景监管相对宽松，但技术本身属于敏感范畴。
>     

> - **生物特征分类**：如果你通过声纹推断成员的敏感属性（如宗教信仰、政治取向等），这属于**高度敏感**的推断。
>     

> - **年龄检测**：如果系统通过声纹自动推断小孩的精确年龄以进行内容过滤，虽然技术上可行（如 WavLM 或特定的年龄估计模型），但在涉及未成年人数据处理时，合规要求会显著提高
>     

  

细节见 https://ai-act-service-desk.ec.europa.eu/en/ai-act/annex-3?

如果项目被认定为“高风险 AI”，对应的审查成本较高。其核心流程可以概括为：**进行“算法影响评估（AIA）”，建立“质量管理体系（QMS）”，完成“技术文档备案与合规性自测”，并在产品全生命周期内持续执行“人类监督”与“风险记录审计”，最终向监管机构提交《欧盟符合性声明（EU Declaration of Conformity）》并加贴“CE认证标识”。**

#### 3.1.2 纯端侧方案

**端侧的要求是足够轻量化，从推理器和模型两个角度考虑轻量化**

CAM++ 采用密集连接的时延神经网络（Densely Connected TDNN）架构，在保持极度轻量化的同时，能够显著提升局部特征的重用率与全局语境的建模能力；该模型原生支持 PyTorch 开发，并能通过 ONNX 导出，完美兼容 TensorRT 加速引擎，是嵌入式端侧**实现高性能声纹识别（SV/SID）与实时说话人确认**的首选方案。（该方案无法实现年龄以及情感识别）

##### **方案一：sherpa-ncnn + CAM++（推荐）**

Sherpa-ONNX 是由新一代 Kaldi（k2-fsa 团队）开发的一个全能型语音处理部署框架。它最大的特点是将特征提取（Kaldi-native-fbank）和神经网络推理（ONNX Runtime）无缝集成在了一起，让开发者可以实现“原始音频输入，识别结果输出”。

##### 方案二：**Kaldi + onnxruntime+CAM++（优势在于 onnxruntime 推理器的复用）**

**Kaldi + ONNX Runtime** 方案的核心逻辑是将 **Kaldi** 强大的传统信号处理能力（如 VAD 静音检测、能量归一化）与 **ONNX Runtime** 跨平台的高效推理引擎相结合。开发者利用 Kaldi 算子完成底层声学特征（Fbank/MFCC）的提取，随后将特征流输入到经过量化优化的 **CAM++** ONNX 模型中，从而在保证识别精度的前提下，实现计算资源极度受限环境下的毫秒级实时响应。

##### 两种方案对比

|   |   |   |
|---|---|---|
|维度|方案一：Sherpa-ONNX + CAM++|方案二：Kaldi + ONNX Runtime + CAM++|
|集成难度|极低（开箱即用，内部封装了所有 C++ 逻辑）|中等（需手动对接音频特征库与推理引擎接口）|
|依赖体积|极轻量（二进制文件极小，适合嵌入式/移动端）|较重（Kaldi 静态库体积较大，编译较复杂）|
|处理流式语音|原生支持流式识别，延迟极低|需自行维护音频 Buffer 和状态控制|
|定制化空间|相对固定，主要依赖框架更新|极高（可深度定制前端降噪、VAD 等预处理算法）|
|适用场景|快速落地、APP集成、资源敏感型边缘设备|专业音频硬件、对信号预处理有特殊要求的系统|

2. 声纹服务进行推理、特征比对
    
3. 返回JSON字段给user进程，返回的JSON格式见 3.1.1

#### 3.1.2 云端协同方案

云端协同方案的思路是在端侧对敏感数据进行处理。如声音，可以在边缘端提取出对应的特征然后利用云端的算力进行推理。该方案将声纹识别链路进行物理拆分—— 在端侧完成模拟信号到数字特征的转化，而在云端完成高复杂度的模型推理与大规模库比对。这种架构在保护用户隐私的同时，缓解了边缘设备在运行重型模型（如 WavLM）时的功耗与算力瓶颈。

#### 实现细节：

- **端侧任务（边缘提取）：** 利用 **CAM++** 等轻量化模型或 Kaldi 特征算子，在本地完成语音采集、VAD（静音检测）以及特征向量（Embedding）的提取。由于传输的是高度抽象的数学特征而非原始音频流，从源头上降低了声音泄露的风险。
    
- **云端任务（深度推理）：** 云端部署高性能 **WavLM (Transformer)** 模型或大型声纹库。接收到端侧传回的特征向量后，利用 GPU 集群进行复杂的非线性变换、去噪补偿以及 1:N 的大规模检索比对。
    
- **协同机制：**
    
    - **动态负载均衡**：当网络环境良好时，由云端执行高精度识别；当网络中断时，端侧自动降级为本地简易模型处理。
        
    - **隐私增强（Privacy-Preserving）**：通过特征脱敏技术，确保即便特征数据被截获，攻击者也无法通过“逆向工程”还原出用户的原始人声。

### 3.2 性能指标预期

无法获取竞品数据，示例如下

|   |   |   |   |
|---|---|---|---|
|指标|目标值|备注|**用户感受**|
|**EER (等错误率)**|$$\le 5\%$$|理想实验环境下|极难被他人或录音伪造冒用。|
|**FAR (误识率)**|$$\le 0.05\%$$|金融级安全标准|主人正常说话，识别失败的情况极少发生。|
|**FRR** **(拒识率)**|$$\le 2\$$|保证用户体验|属于目前工业级轻量化模型的顶尖水平。|
|**响应耗时**|$$\le 500m$$|含网络传输与特征比对|感觉不到延迟，达到“话落即解锁”的体验。|

**FAR (误识率)**：指陌生人被系统错误识别为合法用户的概率，数值越低系统越安全。

**FRR** **(拒识率)**：指本人说话却被系统拒绝识别的概率，数值越低系统越好用。

**EER (等错误率)**：指误识率和拒识率达到平衡时的数值，是评价声纹模型好坏的核心“分数”。

**响应耗时**：**反应速度**，指从你话音落下到系统给出识别结果之间的时间，决定了用户是否感到“卡顿”。
