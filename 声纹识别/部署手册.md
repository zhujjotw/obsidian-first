## 🚀 快速部署

### 前置要求

- **服务器**：Ubuntu 22.04 LTS
- **GPU**：NVIDIA A10G (24GB) × 2-4
- **CUDA**：12.1+
- **Docker**：24.0+
- **内存**：每节点 ≥ 32GB
- **存储**：SSD 500GB+

### 1. 安装 NVIDIA Driver

```bash
# 添加 NVIDIA 仓库
distribution=$(. /etc/os-release;echo $ID$VERSION_ID)
curl -s -L https://nvidia.github.io/nvidia-docker/gpgkey | sudo apt-key add -
curl -s -L https://nvidia.github.io/nvidia-docker/$distribution/nvidia-docker.list | \
  sudo tee /etc/apt/sources.list.d/nvidia-docker.list

# 安装驱动
sudo apt-get update
sudo apt-get install -y nvidia-driver-535

# 重启
sudo reboot

# 验证
nvidia-smi
```

### 2. 安装 Docker & NVIDIA Container Toolkit

```bash
# 安装 Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# 安装 NVIDIA Container Toolkit
distribution=$(. /etc/os-release;echo $ID$VERSION_ID)
curl -s -L https://nvidia.github.io/nvidia-docker/gpgkey | \
  sudo apt-key add -
curl -s -L https://nvidia.github.io/nvidia-docker/$distribution/nvidia-docker.list | \
  sudo tee /etc/apt/sources.list.d/nvidia-docker.list

sudo apt-get update
sudo apt-get install -y nvidia-container-toolkit

# 配置 Docker
sudo nvidia-ctk runtime configure --runtime=docker
sudo systemctl restart docker

# 验证
docker run --rm --gpus all nvidia/cuda:12.1.0-base-ubuntu22.04 nvidia-smi
```

### 3. 部署 Redis

```bash
# 使用 Docker Compose
docker run -d \
  --name redis \
  --restart unless-stopped \
  -p 6379:6379 \
  -v redis_data:/data \
  -e REDIS_MAXMEMORY=2gb \
  -e REDIS_MAXMEMORY_POLICY=allkeys-lru \
  redis:7-alpine

# 验证
docker exec -it redis redis-cli ping
```

### 4. 构建并部署 API 服务

```bash
# 克隆仓库
git clone https://github.com/your-org/voiceprint-cloud.git
cd voiceprint-cloud

# 下载模型
cd models
wget https://huggingface.co/microsoft/wavlm-base-plus/resolve/main/pytorch_model.bin
wget https://huggingface.co/microsoft/wavlm-base-plus/resolve/main/config.json
cd ..

# 复制微调权重
cp /path/to/best_model_eer_0.0350.pt models/checkpoints/

# 构建镜像
docker build -t voiceprint-api:latest -f deployment/Dockerfile .

# 运行容器
docker run -d \
  --name voiceprint-api \
  --restart unless-stopped \
  --gpus all \
  -p 8000:8000 \
  -e REDIS_URL=redis://redis:6379/0 \
  -e NVIDIA_VISIBLE_DEVICES=0 \
  -v $(pwd)/models:/app/models:ro \
  voiceprint-api:latest

# 查看日志
docker logs -f voiceprint-api
```

### 5. 配置 Nginx 反向代理

```nginx
# /etc/nginx/conf.d/voiceprint.conf
upstream voiceprint_backend {
    least_conn;
    server voiceprint-api-1:8000;
    server voiceprint-api-2:8000;
    server voiceprint-api-3:8000;
}

server {
    listen 80;
    server_name api.voiceprint.example.com;

    # 强制 HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name api.voiceprint.example.com;

    # SSL 证书
    ssl_certificate /etc/nginx/ssl/fullchain.pem;
    ssl_certificate_key /etc/nginx/ssl/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    # 请求大小限制 (10MB)
    client_max_body_size 10M;
    client_body_timeout 60s;

    # 代理设置
    location /api/v1/ {
        proxy_pass http://voiceprint_backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # 超时设置
        proxy_connect_timeout 10s;
        proxy_send_timeout 30s;
        proxy_read_timeout 30s;
    }

    # 健康检查
    location /health {
        proxy_pass http://voiceprint_backend/api/v1/health;
        access_log off;
    }
}
```

## 🎯 Kubernetes 部署

### 1. 创建命名空间

```bash
kubectl create namespace ai-services
```

### 2. 部署 Redis

```yaml
# deployment/kubernetes/redis.yaml
apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: redis
  namespace: ai-services
spec:
  serviceName: redis
  replicas: 1
  selector:
    matchLabels:
      app: redis
  template:
    metadata:
      labels:
        app: redis
    spec:
      containers:
      - name: redis
        image: redis:7-alpine
        ports:
        - containerPort: 6379
        resources:
          requests:
            memory: "2Gi"
            cpu: "500m"
          limits:
            memory: "4Gi"
            cpu: "1000m"
        volumeMounts:
        - name: redis-data
          mountPath: /data
  volumeClaimTemplates:
  - metadata:
      name: redis-data
    spec:
      accessModes: ["ReadWriteOnce"]
      resources:
        requests:
          storage: 20Gi
---
apiVersion: v1
kind: Service
metadata:
  name: redis
  namespace: ai-services
spec:
  selector:
    app: redis
  ports:
  - port: 6379
    targetPort: 6379
```

### 3. 部署 API 服务

```yaml
# deployment/kubernetes/deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: voiceprint-api
  namespace: ai-services
spec:
  replicas: 3
  selector:
    matchLabels:
      app: voiceprint-api
  template:
    metadata:
      labels:
        app: voiceprint-api
    spec:
      containers:
      - name: voiceprint-api
        image: your-registry/voiceprint-api:v1.0.0
        ports:
        - containerPort: 8000
        env:
        - name: REDIS_URL
          value: "redis://redis:6379/0"
        - name: LOG_LEVEL
          value: "info"
        resources:
          requests:
            nvidia.com/gpu: 1
            memory: "8Gi"
            cpu: "2000m"
          limits:
            nvidia.com/gpu: 1
            memory: "16Gi"
            cpu: "4000m"
        volumeMounts:
        - name: model-cache
          mountPath: /app/models
        livenessProbe:
          httpGet:
            path: /api/v1/health
            port: 8000
          initialDelaySeconds: 60
          periodSeconds: 30
        readinessProbe:
          httpGet:
            path: /api/v1/health
            port: 8000
          initialDelaySeconds: 30
          periodSeconds: 10
      volumes:
      - name: model-cache
        emptyDir: {}
---
apiVersion: v1
kind: Service
metadata:
  name: voiceprint-service
  namespace: ai-services
spec:
  selector:
    app: voiceprint-api
  ports:
  - port: 80
    targetPort: 8000
  type: ClusterIP
```

### 4. 配置 HPA (自动扩缩容)

```yaml
# deployment/kubernetes/hpa.yaml
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: voiceprint-hpa
  namespace: ai-services
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: voiceprint-api
  minReplicas: 2
  maxReplicas: 10
  metrics:
  - type: Resource
    resource:
      name: cpu
      target:
        type: Utilization
        averageUtilization: 70
  - type: Resource
    resource:
      name: memory
      target:
        type: Utilization
        averageUtilization: 80
```

### 5. 应用配置

```bash
# 应用所有配置
kubectl apply -f deployment/kubernetes/redis.yaml
kubectl apply -f deployment/kubernetes/deployment.yaml
kubectl apply -f deployment/kubernetes/hpa.yaml
kubectl apply -f deployment/kubernetes/ingress.yaml

# 检查状态
kubectl get pods -n ai-services
kubectl get svc -n ai-services
kubectl get hpa -n ai-services

# 查看日志
kubectl logs -f deployment/voiceprint-api -n ai-services
```

## 🚨 故障排查

### 常见问题

| 问题 | 解决方案 |
|-----|---------|
| GPU 不可用 | 检查 NVIDIA Driver 和 nvidia-container-toolkit |
| 内存溢出 | 减少 batch_size 或增加 GPU 内存 |
| 响应慢 | 检查网络延迟、GPU 利用率 |
| 502 错误 | 检查后端服务健康状态 |
| EER 升高 | 检查模型版本、数据分布变化 |

### 日志查看

```bash
# Docker 日志
docker logs -f voiceprint-api

# Kubernetes 日志
kubectl logs -f deployment/voiceprint-api -n ai-services

# 查看所有 Pod 日志
kubectl logs -l app=voiceprint-api -n ai-services --all-containers=true
```
