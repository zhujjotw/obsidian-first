# 转生产设计联系书创建工具

## 项目概述

**项目名称**: pm_create_factory_contact_doc
**主要功能**: 自动化创建海信电视产品转生产设计联系书的工具，通过FastAPI提供REST API服务

---

## 完整业务流程

### 流程图

```
用户输入（项目名称/测试申请单号/继承联系书名称）
    ↓
验证参数有效性（项目名称和测试申请单不能同时为空）
    ↓
确定继承联系书（从contact_info中提取或查询该机型最新联系书）
    ↓
检查项目是否存在（调用PLM接口）
    ↓
处理测试申请单（未提供则查询最新测试申请单，提供则验证归属）
    ↓
初始化测试申请单数据（获取原型机、市场、平台、版本等关键信息）
    ↓
检查是否已存在联系书（对比测试版本避免重复创建）
    ↓
确定继承关系
    ├─ 首次创建：指定联系书/同系列联系书/原型机联系书
    └─ 非首次创建：本机型最新联系书
    ↓
生成SVN软件地址（首次创建根据规则生成新路径，非首次创建基于历史路径递增编号）
    ↓
获取FTP文件列表（连接FTP服务器获取ci_output和OTA_INC目录文件列表）
    ↓
收集联系书数据（调用BaseService的80+个getter方法收集所有字段数据）
    ↓
打包数据（组装成PLM API所需格式）
    ↓
创建联系书（调用PLM API创建联系书获取名称、ID、URL）
    ↓
记录操作日志到数据库
    ↓
返回结果（联系书名称、ID、URL）给用户
```

### 单行流程描述

**用户输入>验证参数有效性>确定继承联系书>检查项目是否存在>处理测试申请单>初始化测试申请单数据>检查是否已存在联系书>确定继承关系>生成SVN软件地址>获取FTP文件列表>收集联系书数据>打包数据>创建联系书>记录操作日志>返回结果给用户**

## 核心类: BaseService

### 主要方法分类

#### 项目验证方法
- `is_project_exist()` - 检查项目是否存在
- `inquire_request_exist_contact_letter()` - 查询测试申请单是否已有联系书
- `inquire_series_exist_contact_letter()` - 查询同系列机型的联系书
- `inquire_latest_test_request()` - 查询最新测试申请单

#### 数据初始化方法
- `init_test_request_data()` - 初始化测试申请单数据
- `init_contact_letter_data()` - 初始化联系书数据
- `init_version_access_path_file_list()` - 初始化版本访问路径文件列表

#### SVN路径生成
- `generate_svn_software_address()` - 生成SVN软件地址
- `get_v_part_number()` - 获取企业项目编号
- `get_product_brand()` - 获取产品品牌

#### 数据获取方法 (80+个getter)
- `get_test_request_version()` - 获取测试版本
- `get_market()` - 获取市场信息
- `get_chip()` - 获取芯片方案
- `get_emmc_file_name()` - 获取EMMC文件名
- `get_ota_file_name()` - 获取OTA文件名
- `get_mboot_file_name()` - 获取Mboot文件名
- `get_usb_file_name()` - 获取USB文件名
- ...等85个字段

## 请求参数说明

### 输入参数 (ContactDocRequest)

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| user_name | str | 是 | 用户名称 |
| ldap_name | str | 是 | LDAP账号 |
| project_name | str | 条件 | 项目名称（与test_request_name至少填一个） |
| test_request_name | str | 条件 | 测试申请单名称（与project_name至少填一个） |
| contact_letter_name | str | 否 | 继承的联系书名称 |

### 输出结果 (ContactDocResponse)

| 字段 | 类型 | 说明 |
|------|------|------|
| name | str | 联系书名称 |
| id | str | 联系书ID |
| url | str | 联系书URL |
| message | str | 处理结果信息 |

## 继承关系规则

### 首次创建场景
1. **指定联系书** → 继承指定的联系书
2. **同系列联系书** → 继承同系列机型的联系书
3. **原型机联系书** → 继承原型机的联系书

### 非首次创建场景
1. 继承本机型最新联系书
2. 或继承指定的联系书
